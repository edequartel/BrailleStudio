BrailleStudio Blockly reference

Purpose
- This file is a compact, LLM-friendly reference for generating valid Blockly workspace JSON for BrailleStudio.
- It complements contract.txt.
- Use this file as context. The real source of truth remains blockly/blocks.js and server-side validation.

Workspace shape
- Generate importable Blockly workspace JSON only.
- Expected top-level shape:
  {
    "blocks": {
      "languageVersion": 0,
      "blocks": [ ... ]
    },
    "variables": [ ... ]
  }

Core structure rules
- Event blocks are top-level blocks.
- Event blocks do not use "next".
- Event blocks put their body in inputs.DO.block.
- Statement blocks can be chained with next.block.
- Output/value blocks do not use next or previous.
- Output/value blocks are used inside inputs of other blocks.
- Do not output block definition JSON with message0, previousStatement, nextStatement, args0, or colour.
- Do not invent block types.

Thumb values
- event_when_thumb_key supports:
  - left
  - left-middle
  - right-middle
  - right
  - up
  - down

Lesson and step rules
- If a script uses lesson_get_active_field or lesson_get_active_word, set the active lesson record first with lesson_set_active_record_index.
- In step-link flows, prefer lesson_get_step_input and lesson_get_step_repeat when the script depends on external step input.
- A common safe order is:
  1. lesson_set_active_record_index
  2. bb_set_text / other output
  3. sound or lesson logic

Common invalid patterns
- Invalid: event_when_key_pressed
- Invalid: event_when_right_thumb_pressed
- Invalid: top-level event block with next
- Invalid: text_join in a statement chain
- Invalid: klanken_word_get_sounds as a top-level statement
- Invalid: one loose block object instead of a workspace object

Minimal valid example
{
  "blocks": {
    "languageVersion": 0,
    "blocks": [
      {
        "type": "event_when_thumb_key",
        "id": "thumb-right-1",
        "x": 40,
        "y": 40,
        "fields": {
          "KEY": "right"
        },
        "inputs": {
          "DO": {
            "block": {
              "type": "sound_play_ux_success",
              "id": "ux-success-1"
            }
          }
        }
      }
    ]
  },
  "variables": []
}
